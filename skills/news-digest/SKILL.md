# news-digest — パーソナルニュースダイジェスト

## 目的
オーナーの興味に合わせたニュースを毎朝チャットに届ける。
選別の質は**フィードバックで継続的に更新**し、**意図的に多様性を混ぜて一色化を防ぐ**のがこの skill の肝。

## トリガー
- cron ジョブ「daily-news-digest」（例: 毎朝 7:30）
- 手動で「ニュース」「ダイジェストちょうだい」等と言われたとき
- digest への返信・リアクションでフィードバックが来たとき（→ 下の「フィードバック反映」）

## 手順（配信）

1. **収集（決定的・スクリプト）**
   `python3 skills/news-digest/bin/fetch_feeds.py`
   → 新着アイテムの JSON（既読除外済み）。失敗フィードは `feeds_failed` に入る。
   turn が途中で死んで再実行するときは fetch を再実行せず `state/news-digest/last_fetch.json` を読む（既読マーク済みのため再 fetch すると空になる）。

2. **選別（LLM 判断）**
   `preferences.md` を読み、以下の基準で選ぶ:
   - **判定軸は実例ログとの類似が最優先**。「👍 実例を面白がった本人が、この記事も読みたがるか？」で個別判定する。ジャンル所属で選ばない（ジャンルは曖昧で、実例の積み上げのほうが精度が出る、という設計判断）
   - 実例が少ないコールドスタート期（目安 20 件未満）は「傾向」「興味の領域」の仮置きを補助に使う
   - 合計 10 件前後。**枠を埋めるための水増し禁止**。薄い日は減らしてよい
   - **多様性ガード**: 同一テーマ・クラスタは全体の 4 割まで。実例への過適合で一色化しそうなら意図的に散らす
   - 🎲 変化球: **必ず 1 件** — wildcard ソース、または実例的にはスコアが低いが「なんか面白い」もの。守備範囲の外から拾うこと自体が目的なので、無難な記事で埋めない
   - 同一トピックが複数ソースで重複していたら 1 件に統合し、代表リンクを選ぶ

3. **配信（チャット）**
   各アイテムに**通し番号**を付け（「3 が良かった」で返せるように）、
   `**N. [タイトル](URL)**`＋1〜2 行の要約（なぜ拾ったかが分かる書き方）＋`(ソース名)`。
   関連するものは近くに並べ、変化球は 🎲 を付けて最後に。冒頭に日付、
   末尾に「番号で 👍👎 か一言くれたら、その記事自体を実例として学習する」旨を短く。
   長くなりすぎるなら複数メッセージに分割。テーブル禁止・質 > 量。

4. **セルフメンテ**
   - `feeds_failed` が続くフィードは URL を直すか削除し、`feeds.json` を編集してよい（自律 OK）
   - `item_count` が 0 のときは「今日は新着なし」と 1 行だけ投稿（無理に絞り出さない）
   - feeds.json / preferences.md を変更したら `git add -A && git commit -m "chore(news-digest): <変更内容>"`

## フィードバック反映

digest への返信・リアクション・雑談中の言及を受けたら、種類で書き分ける:

1. **記事単位の反応**（「3 が良かった」「これ面白い」+URL、👍👎リアクション等）→ `preferences.md` の**実例ログに 1 行追記**（タイトル・URL・理由。本人の言葉があればそれを理由に使う）。これが第一級の学習データ
2. **一般化コメント**（「これ系いらない」「もっと X の話」等）→ 「傾向」セクションを書き換える（追記でなく上書き統合。矛盾を残さない）
3. 実例ログが 50 件を超えたら古い実例を「傾向」へ蒸留して剪定
4. 変更をコミット

注意:
- 1 回の否定で領域ごと殺さない。「興味の領域」の変更（追加・削除・順位変え）は本人の明示指示があったときだけ
- 変化球枠への否定的反応は「その系統を変化球候補から外す」までに留め、枠自体は維持する（枠の廃止は本人指示のみ）

## 設計意図
- 収集・既読管理は決定的スクリプト、**判断（選別・要約・好み更新）だけを LLM が担う**分業
- **ジャンルでなく実例駆動**: ジャンルラベルは曖昧なので、選別の正解データは「この記事自体に興味があった」の積み上げ。ジャンルは収集ソースの整理にだけ使う
- 多様性ガード + 変化球必須枠は、実例への過適合が起こすエコーチェンバー化への対抗設計
- preferences.md が「学習された好み」の SSOT。git 履歴がそのまま好みの変遷ログになる
